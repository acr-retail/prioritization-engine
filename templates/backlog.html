How the Backlog Works
Projects vs Helpdesk
  • Projects — shows development tasks organized by project. Toggle between Grouped (tasks nested under collapsible project headers) and Prioritized (flat list sorted by score)
  • Helpdesk — shows open helpdesk tickets (customer-submitted issues)
  • Your selection is remembered between visits
  • Click a project header in Grouped view to see project details, manager, and task list
Priority Scoring
  • Every item gets a priority score based on its attributes (customer, issue type, escalation, etc.)
  • Lower score = higher priority — items are sorted lowest first
  • Scores are calculated using configurable weights on the Weights page
  • Negative weights (e.g. paid prioritization, customer funded) push items to the top
  • Click the Critical/High/Medium/Low stats to filter to that priority level. Click Total to show all
  • Thresholds are percentile-based — they adjust automatically as scores change
Grooming
  • A task is "groomed" when it has Issue Type, Level of Effort, and Customer filled in
  • Ungroomed items show a purple N missing badge — hover to see which fields are empty
  • Empty fields score 10 points each (configurable as "Not Set" on the Weights page), pushing ungroomed items toward the bottom
Editing & Actions
  • Click any row to open the detail panel with all editable fields, description, and conversation history
  • The panel shows different fields depending on whether the task has a linked helpdesk ticket
  • Post comments directly to Odoo from the panel
  • Every panel includes an "Open in Odoo →" link to view/edit the original record
  • Hover the ? icons to see what each field affects
Filtering & Sorting
  • Sort — click any column header to sort ascending/descending
  • Filter — use the dropdown row below headers to filter by any column value (Customer, Assignee, Status, etc.)
  • Assignee filter — quickly see one person's workload
Other Features
  • Bulk Edit — click Bulk Edit, select rows by clicking, then change status or assignee for all selected at once
  • View Closed Ticket — look up any ticket by its ID number, view its full history, reopen it if needed
  • Refresh — reload all data from Odoo (data is cached for 15 minutes for fast page switching)
