How the Prioritized Backlog Works
Priority Scoring
  • Every task gets a priority score based on its attributes (customer, issue type, escalation, etc.)
  • Lower score = higher priority — tasks are sorted lowest first
  • Scores are calculated using configurable weights on the Weights page
  • Negative weights (e.g. paid prioritization, customer funded) push items to the top
  • Score thresholds for Critical/High/Medium/Low are calculated from the actual distribution of all task scores (percentile-based)
Grooming
  • A task is "groomed" when it has Issue Type, Level of Effort, and Customer filled in
  • Ungroomed tasks show a purple N missing badge — hover to see which fields are empty
  • Empty fields score 10 points each (configurable on the Weights page as "Not Set"), pushing ungroomed items toward the bottom
  • Show Ungroomed filters to only show tasks that need grooming
  • Hide Ungroomed hides ungroomed tasks so you see only actionable items
Editing Tasks
  • Click a cell to edit it inline — dropdowns for selections, text input for title
  • Changes save to Odoo automatically and the score recalculates
  • Click the Score or ID to open the full task editor panel with all fields, description, and conversation
  • The editor panel also lets you post comments directly to the Odoo task
  • Hover the ? icons in the editor to see what each field affects
Other Features
  • Sort — click any column header to sort ascending/descending
  • Filter — use the dropdown row below headers to filter by any column value
  • Bulk Edit — enter bulk mode, click rows to select, then change status or assignee for all selected at once
  • View Closed Ticket — look up a closed ticket by its ID number, view its history, and reopen it if needed
  • Refresh — reload all task data from Odoo
