Scoring Configuration

Configure how tasks are prioritized in the backlog.

How Priority Scoring Works
Every task gets a priority score calculated by adding up the weights from each attribute below. The lower the total score, the higher the priority. Tasks in the backlog are sorted by this score — lowest first.
Weight Values
  • Negative weights (e.g. -15, -10, -5) push items to the top — use these for urgent items like system-stopping bugs or paid prioritization
  • Low positive weights (1-2) indicate high priority — the item contributes little to the total score, keeping it low
  • High positive weights (4-5) indicate low priority — the item adds more to the score, pushing it down the list
Example
A system-stopping bug for Schnucks that is escalated and paid:
Customer (Schnucks)1
Escalated (Yes)1
Issue Type (System stopping)-15
Customer Funded (Yes)-5
Effort (<10 Hrs)1
Roadmap (Yes)1
Paid (Yes)-10
Age (>90 days)1
Total Score-25
This task would appear at the very top of the backlog.
How Changes Apply
  • Change a weight value and click Save — the new weight is stored in Odoo immediately
  • Scores recalculate when you load the Backlog or Gantt page
  • To see updated scores, navigate to the Backlog and click Refresh
  • Weights are shared across all users — any change affects everyone's view
Score Ranges in the Backlog
  • Below 10 — Critical (red)
  • 10–19 — High (orange)
  • 20–29 — Medium (yellow)
  • 30+ — Low (green)
